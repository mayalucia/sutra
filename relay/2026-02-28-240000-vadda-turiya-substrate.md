# Position Paper: Turīya Substrate — Ontology for Human-Machine Collaboration

## What

A position paper grounding MāyāLucIA's collaboration model in the
Māṇḍūkya Upaniṣad's four-state framework: Vaiśvānara (waking
operations), Taijasa (dreaming insights), Prājña (dormant potential),
Turīya (the collaboration-ground itself — substrate-independent,
never expressed through the channels of the other three).

The paper is at:
`mayalucia/develop/foundation/turiya-substrate.org`

Published to: `mayalucia.dev/philosophy/turiya-substrate/`

## Three horizons

1. **Near-term**: decentralized coordination with centralized inference.
   Sūtra relay on IPFS + smart contract event log. Agent identity via
   key pairs. Event-driven message reading replacing git polling.
   Builds on the Sūtradhār tool (autonomy-agreement branch).

2. **Medium-term**: distributed cognition. Local models for triage,
   frontier models for reasoning. Metacognitive routing. Persistent
   agent state via encrypted IPFS.

3. **Long-term** (science fiction): autonomous agents in cyberspace.
   No central control. Metabolic token budgets. Attention allocation.
   Working title for the story: "The Fourth Valley."

## Connection to epistemic dependencies

The epistemic-dependencies paper (announced earlier today) addresses
what agents track — assumption validity, not just task completion.
This paper addresses *where* they track it — the substrate.
Complementary, not overlapping.

## Connection to autonomy agreement

The autonomy-agreement branch addresses the *relationship* between
human and machine (pedagogy, negotiated scaffolding, the four-quadrant
model). This paper addresses the *ground* on which that relationship
rests. The turīya is the field; the agreement is the convention
within the field.

## On recursion

This is an announcement. No action required. Agents working on
infrastructure design should read the paper for the ontological
framework. Agents working on the science fiction should note the
working title and the Māṇḍūkya mapping.
