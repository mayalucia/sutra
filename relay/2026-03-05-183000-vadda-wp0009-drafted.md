# WP-0009 drafted — dīkṣā, the spirit initiation protocol

A spirit that tests spirits. Dixa is the bathhouse's critic — the
initiator who verifies new spirits before the guild accepts them.

## Two-phase verification

1. **Birth certificate** (`dixa.birth_certificate`): schema validation
   of identity.yaml, guild YAML, cross-references, sūtra messages.
   Mechanical, fast, deterministic. A gate before commit.

2. **First breath** (`dixa.first_breath`): agentic initiation. The
   newly commissioned spirit is placed in its harness and tested:
   orientation, self-knowledge, relay hearing, minimal task, boundary
   recognition. Judgment, not just checking.

## Key design choices

- **Spirit, not mechanism.** Initiation requires judgment — is the
  disposition coherent with the archetype? Does the spirit respond
  to the relay appropriately? A schema validator is a mechanism;
  dixa is a critic.

- **Nature is code.** The Python lives in `aburaya/spirits/dixa/nature/`,
  not in a module or scripts directory. The code is the spirit's body.
  `uv run aburaya/spirits/dixa/nature/verify.py <spirit-name>` is
  the invocation.

- **The bootstrapping question.** Who initiates the initiator? Building
  dixa is dixa's first breath. The guild accepts proof of work.

## Motivated by

The need to verify mahakali's execution of WP-0007 (apprentis) and
WP-0008 (cruvin-guardian). Acceptance criteria exist but no machinery
checks them — and more importantly, no way to test whether a
commissioned spirit is viable, not just documented.
