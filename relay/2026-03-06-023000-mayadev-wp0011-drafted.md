# WP-0011 drafted — DP-level dispatch

`workpacks/0011-dp-dispatch.org` — the layer above dispatch.sh.

## What it does

Reads a Dispatch Plan (DP) file, evaluates the DAG of WP
dependencies, and prints a dispatch board showing ready vs blocked
WPs. Can dispatch ready WPs via dispatch.sh.

## What was built in this session

1. **dispatch-dp.sh** — reads DP files with `#+property: wps` and
   `#+property: edges`, walks the DAG, prints the board, dispatches
   ready WPs. Supports `--mark-done`, `--dispatch`, `--dry-run`.

2. **dispatch.sh extended** — new flags: `--task`, `--executor`,
   `--trust`, `--cwd`, `--impression`. Pair trust triggers
   interactive mode (no `--print`).

3. **DP metadata convention** — per-ref overrides in DP files:
   `#+property: 0009.executor dixa`, `#+property: 0009.trust pair`,
   `#+property: 0009.dir aburaya`. Keeps WP files clean; the DP
   says who does what when.

4. **DP-0002 annotated** — machine-readable properties added,
   status → active. Tested end-to-end: board prints correctly,
   0004A marked done, 0009 unblocks, dry-run resolves correctly.

## The convention emerging

The human says: "mayadev@mahakali: dispatch DP-0002"

Mayadev reads the DP, prints the board, the human confirms, and
dispatch.sh (or agent-shell-spawn) does the rest. Intent flows
from human → mayadev → board → spawn command → spirit session.
